    # 一. 定义

    在es6之前，变量声明用var,会存在变量的预解析，预解析会执行在当前作用域的最顶部，作用域有三种

    • 全局作用域
    • 块级作用域
    • 函数作用域

    例子:

    console.log(xiaoqingb)
var xiaoqingb = 666
console.log(xiaoqingb)
add(1 ， 2)
function add(x , y){
	var total = x + y
	console.log(total)
}
// 输出 undefined 666 3
    1. # 先执行预解析

      全局作用域下，xiaoqingb = undefined ,add = function scoped->null

      函数add作用域下，total = undefined, scoped ->全局作用域

    2. # 再执行命令操作

      第一行命令输出变量xiaoqingb，此时xiaoqingb = undefined ，所以打印出undefined

      第二行命令给变量赋值，所以xiaoqingb变成了666

      第三行因此输出666

      第四行调用函数,进入函数作用域

      函数第一行给total赋值，这时候total = 3

      函数第二行直接打印 3

    # 作用域链

    # 定义

    如果在一个作用域里调用一个变量，这个变量声明并不存在当前作用域里，则会向外层作用域找是否有这个变量的声明，直到全局作用域，

    如果找到了，就会直接调用该变量。

    如果找不到，就会报错。

    例子:

    console.log(total)     // 1 (行数)
var total = 0		   
function add(x , y){   // 3
	console.log(total)
	var total = x + y  // 5
	console.log(total)
}					   // 7
add(1 , 2)
console.log(total)	   // 9
// 输出 0  undefined 3  0
    1. # 先进行预解析

      全局作用域: total = undefined add = function ,scoped->null

      函数add作用域: total = undefined ，scoped ->全局作用域

    2. # 执行命令

      第一行由于此时total = undefined,同样是打印出undefined

      第二行给total赋值了，这时候total变成了0

      第三到第七是函数声明还未调用

      第八行调用函数，进入函数作用域

      函数第一行打印total这个变量 , 当前作用域下total已存在，值为undefined，所以输出undefined

      函数第二行把函数作用域里的total变成了 3

      函数第三行则输出3，跳出函数

      第九行由于当前作用域存在total，且值为0，所以输出了0

    例子:

    console.log(total)     // 1 (行数)
var total = 0		   
function add(x , y){   // 3
	console.log(total)
	total = x + y      // 5  
	console.log(total)
}					   // 7
add(1 , 2)
console.log(total)	   // 9
// 输出 undefined  0  0  3  3

    此时取消了函数里的 var关键字，total就不是函数内的私有变量了.

    # 总结

    1. 全局作用域下有var 和 无var的区别

      console.log(a) // undefined
var a = 111
console.log(b) // 报错
b = 12

    2. 在任何作用域下，只要有var声明，都会提前，无视判断条件！！！

      if(false){
	var a = 0
}
console.log(a) // undefined

    3. 只预解析“=”左边的，右边的是指，不参与预解析

      fn(); // -> undefined();  // Uncaught TypeError: fn is not a function
var fn = function () {
    console.log('ok');
}

fn(); -> 'ok'
function fn() {
    console.log('ok');
}
fn(); -> 'ok'

    4. 立即执行函数：定义和执行一起完成

      ~ function (num) {}(100)  
+ function (num) {}(100)  
- function (num) {}(100)  
! function (num) {}(100)

    5. return下的代码依然会进行预解析

      function fn() {                             
    console.log(num); // -> undefined
    return function () {             

    };                               
    var num = 100;                   
}                                  
fn();

    6. 名字已经声明过了，不需要重新的声明，但是需要重新的赋值

      var fn = 13;                                       
function fn() {                                    
    console.log('ok');                               
}                                                  
fn(); // Uncaught TypeError: fn is not a function

      应该改成fn = function (){....}，才能
